## 🚀 第1步: 上传代码到GitHub

### 1.1 创建GitHub账号 (如还没有)

1. 访问 https://github.com
2. 点击 "Sign up" 注册免费账号
3. 验证邮箱

### 1.2 创建新仓库

1. 登录后,点击右上角头像旁的 "+" 号
2. 选择 "New repository"
3. 填写信息:
   - **Repository name**: `guangzhou-daily-analysis` (或其他您喜欢的名字)
   - **Description**: 广州日报舆情分析系统 (可选)
   - **Public/Private**: 选择 **Public** ⚠️ (必须公开,Streamlit Cloud才能访问)
   - **Initialize this repository with**: 不要勾选任何选项
4. 点击 "Create repository"

### 1.3 上传代码到GitHub

在您的项目文件夹中打开PowerShell (在当前文件夹地址栏输入 `powershell` 并回车),然后依次执行以下命令:

```powershell
# 初始化Git仓库
git init

# 添加所有文件到暂存区
git add .

# 提交代码
git commit -m "Initial commit: 广州日报舆情分析系统"

# 关联GitHub仓库 (请将 您的用户名 替换为您的实际GitHub用户名)
git remote add origin https://github.com/您的用户名/guangzhou-daily-analysis.git

# 设置主分支名称
git branch -M main

# 推送到GitHub (首次推送需要输入GitHub账号密码或使用Token)
git push -u origin main
```

**⚠️ 重要提示:**
- 将命令中的 `您的用户名` 替换为您实际的GitHub用户名
- 如果推送时要求认证,建议使用GitHub Token (https://github.com/settings/tokens)
- 或者使用GitHub Desktop图形化工具更简单

### 1.4 验证上传成功

访问您的GitHub仓库页面,确认能看到以下关键文件:
- ✅ app_final_v3.py
- ✅ news_with_sentiment.csv
- ✅ guangdong_true_mask.png
- ✅ monthly_hot_topics.csv
- ✅ stopwords.txt
- ✅ requirements.txt
- ✅ .gitignore

## ☁️ 第2步: 在Streamlit Community Cloud部署

### 2.1 访问Streamlit Cloud

1. 打开浏览器,访问: https://streamlit.io/cloud
2. 点击 "Sign up for free" 或 "Sign in"
3. 选择 "Continue with GitHub" 授权登录

### 2.2 创建新应用

1. 登录后,点击页面上的 **"New app"** 按钮
2. 填写部署信息:
   - **Repository**: 在下拉框中选择您刚才创建的仓库 (例如: `您的用户名/guangzhou-daily-analysis`)
   - **Branch**: 选择 `main`
   - **Main file path**: 输入 `app_final_v3.py`
3. 点击 **"Deploy!"** 按钮

### 2.3 等待部署完成

- 首次部署需要 **2-5分钟** (需要安装Python依赖包和加载数据)
- 您可以在部署页面看到实时日志
- 部署成功后,会自动跳转到您的应用页面

### 2.4 获取访问链接

部署成功后,您会获得一个永久访问链接,格式为:
```
https://您的应用名.streamlit.app
```

**示例:** `https://guangzhou-daily-analysis.streamlit.app`

## 🔧 常见问题解决

### 问题1: Git推送失败,提示认证错误

**解决方案:**
1. 使用GitHub Token代替密码:
   - 访问 https://github.com/settings/tokens
   - 生成新Token (勾选repo权限)
   - 复制Token,在Git推送时作为密码使用

2. 或使用GitHub Desktop图形化工具 (推荐新手)

### 问题2: 部署时提示缺少依赖

**解决方案:**
- 检查 `requirements.txt` 是否已正确上传到GitHub
- 查看Streamlit Cloud的部署日志,确认具体错误信息

### 问题3: 应用加载很慢

**原因:** 
- 首次加载需要处理6700+条新闻数据
- Streamlit Cloud免费层资源有限

**优化建议:**
- 应用已使用 `@st.cache_data` 缓存,后续访问会快很多
- 如果仍然很慢,可以考虑升级Streamlit Cloud套餐

### 问题4: 中文显示乱码

**解决方案:**
- 确保 `app_final_v3.py` 文件编码为 UTF-8
- Streamlit Cloud已内置中文字体支持,应该没问题
